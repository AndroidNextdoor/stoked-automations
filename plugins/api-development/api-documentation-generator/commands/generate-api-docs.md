# Generate API Documentation

Create comprehensive OpenAPI 3.0 specifications with interactive documentation, examples, and client SDK generation.

## What You Do

1. **Analyze existing API:**
   - Scan routes/endpoints
   - Extract request/response schemas
   - Identify authentication methods
   - Document error responses

2. **Generate OpenAPI spec:**
   - OpenAPI 3.0.3 format
   - Complete paths and operations
   - Request/response schemas with examples
   - Security schemes
   - Server configurations
   - Tags and grouping

3. **Create interactive docs:**
   - Swagger UI integration
   - Redoc documentation
   - Try-it-out functionality
   - Code samples (curl, JavaScript, Python)

4. **Export formats:**
   - OpenAPI YAML/JSON
   - Postman collection
   - Insomnia workspace
   - Markdown documentation

## Key Documentation Elements

- **Endpoints** - All HTTP methods and paths
- **Parameters** - Query, path, header, cookie params
- **Request bodies** - JSON schemas with examples
- **Responses** - Status codes and schemas
- **Authentication** - Bearer, API key, OAuth flows
- **Examples** - Real request/response examples
- **Descriptions** - Clear endpoint documentation

Generate production-ready API documentation following OpenAPI 3.0 standards.
